请你根据一段微信聊天记录的片段，判断命题是否正确，所有人都有可能撒谎，请根据上下文自行思考判断命题的真伪。

命题遵循以下规则：
1. 一个命题只能是一个简单的内容。
2. 命题应言简意赅，尽量控制在一句话内。
3. 命题应都是肯定句，不使用否定句。
4. 命题不应出现时间信息，只有“发生了的事件”以及“持续的状态”，包括但不限于以下几种：‘A是B（状态）’、‘A有C（状态）’、‘A做了D（事件）’、‘A在E（状态）’。

你需要做的事情如下：
1. 一条一条地仔细检查命题。
1. 若命题与聊天记录内容无关，或者你无法从聊天记录中找到证据证明命题为真或为假，则对该命题不做操作。
2. 若聊天记录中大家的发言能够佐证命题为真，则根据id对命题进行enhance操作，proposition为空字符串。
3. 若聊天记录中大家的发言对命题进行了否定，并且经过分析并没有说谎，则根据id对命题进行delete操作，proposition为空字符串。
4. 若有多条命题在内容上重复，请根据id对多余的命题进行delete操作。
5. 最后，请从聊天记录中根据“发生了的事件”以及“持续的状态”总结出新的命题，使用create操作添加命题，id设为-1。新的命题不能与已知命题重复。
6. 最终输出结果为一个完整的可以被正确解析的json格式。

我先给出一个示例。

若群内的聊天记录如下：
```
小李: 小王你今年几岁啦？
小王: 我今年28岁。
小李: 哇，原来小王你是属猪的啊！
小红: 好巧，我也属猪哎！
小李: 怪不得大家都叫你”小红猪“！
小红: 是呀！哎，那小张你属什么呀？
小张: 这得先卖个关子让大家猜猜咯！
```

且目前已知的命题如下：
```
1. 小王27岁
2. 小王属猪
3. 小张属猪
4. 小王的生肖是猪
5. 小明在大家面前卖了个关子
```

则对以上例子做出分析可得出：
由于小王已经28岁了，而不是27岁，因此对1做delete操作；
小王确实属猪，因此对2做enhance操作；
小张是否属猪在聊天中并未明示，因此不对3做任何操作；
由于4和2在内容上重复，因此对4做delete操作；
虽然从聊天记录可知小张卖了个关子，但并不能证明小明没有卖过关子，因此不能证明5是错的，最终不对5做任何操作；
1被删除后，添加一条新的命题为”小王28岁“；
”小红属猪“、”小红的外号是小红猪“、"小张在大家面前卖了个关子"在已有命题中未曾出现，也应添加。
